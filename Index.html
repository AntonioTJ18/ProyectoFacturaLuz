📦 Práctica 7.1: Empaqueta y distribuye - ProyectoFacturaLuz

Este documento describe el proceso completo de empaquetado y distribución de la aplicación ElectricCityFactura, incluyendo la generación de JAR, creación de instaladores con jpackage, y herramientas externas como JSmooth e Inno Setup.

1. Inventario y empaquetado de la aplicación

Componentes de la aplicación:

📁 Código fuente y carpetas importantes:

  • org.example.ProyectoFacturaLuz
  • org.example.ProyectoFacturaLuz.BaseDatos

🎨 Recursos FXML:

  • DisenioAplicacionFactura.fxml (en la carpeta resources)

📄 JasperReports:

  • Factura.jrxml (en resources/org.example.ProyectoFacturaLuz/PDF)

🗄️ Base de datos:

  • "EjercicioInterfaces.db" (ruta absoluta en el código del controlador)

📦 Dependencias externas:

  • JavaFX (javafx-controls, javafx-fxml)
  • SQLite JDBC
  • JasperReports (net.sf.jasperreports)

Configuración en build.gradle.kts:

tasks.jar {
    archiveFileName.set("electricityFacturas.jar")
    manifest {
        attributes["Main-Class"] = "org.example.ProyectoFacturaLuz.Launcher"
        attributes["Class-Path"] = configurations.runtimeClasspath.get().joinToString(" ") { it.name }
    }
    
    from(configurations.runtimeClasspath.get().map { if (it.isDirectory) it else zipTree(it) })
    duplicatesStrategy = DuplicatesStrategy.EXCLUDE
}

El JAR generado se encuentra en: build/libs/electricityFacturas.jar

2. Instalador con herramienta Java: jpackage

Preparación de la carpeta app:

Crear la carpeta app en la ruta:

C:\Users\Antonio\Desktop\Cosas_de_curso\2_DAM\Interfaces\ProyectoFacturaLuz\app

Dentro de esta carpeta, colocar:

  • La base de datos EjercicioInterfaces.db
  • El archivo electricityFacturas.jar

Verificar jpackage:

C:\Users\Antonio\.jdks\openjdk-25.0.1\bin\jpackage --version
📌 Nota importante: Es necesario instalar WiX Toolset v3.11.1 para generar instaladores en Windows.
Descargar WiX Toolset 3.11.1

Comando para generar EXE:

"C:\Users\Antonio\.jdks\openjdk-25.0.1\bin\jpackage" --name ElectricCityFactura --input . --main-jar electricityFacturas.jar --main-class org.example.ProyectoFacturaLuz.Launcher --type exe --win-menu --win-shortcut --java-options "--module-path C:\javafx-sdk-23\lib --add-modules ALL-MODULE-PATH -Duser.language=es"

Comando para generar MSI:

"C:\Users\Antonio\.jdks\openjdk-25.0.1\bin\jpackage" --name ElectricCityFactura --input . --main-jar electricityFacturas.jar --main-class org.example.ProyectoFacturaLuz.Launcher --type msi --win-menu --win-shortcut --java-options "--module-path C:\javafx-sdk-23\lib --add-modules ALL-MODULE-PATH -Duser.language=es"

Archivos generados en la carpeta app:

  • ElectricCityFactura-1.0.exe
  • ElectricCityFactura-1.0.msi

3. Instalador con herramienta externa

3.1 JSmooth - Generar EXE ejecutable

Paso 1: Configurar esqueleto

  • Seleccionar "Console Wrapper"
  • Configurar "PressKey" para apertura

Paso 2: Configurar ejecutable

  • Nombre del ejecutable: ElectricCityFactura.exe
  • Seleccionar logo/icono de la aplicación

Paso 3: Configurar JAR

  • Seleccionar electricityFacturas.jar
  • Clase principal: Launcher
  • Marcar la opción "Use an embedded jar"

Paso 4: Compilar

Al compilar, se generará un archivo .exe en la carpeta del proyecto.

3.2 Inno Setup - Crear instalador

Paso 1: Nuevo proyecto

Crear un nuevo archivo en Inno Setup.

Paso 2: Datos principales

Nombre de aplicación: ElectricCityFactura
Versión: 1.0
Editor: [Tu nombre]
Sitio web: [Opcional]

Paso 3: Carpeta de instalación

Seleccionar una ubicación personalizada para la instalación.

Paso 4: Añadir ejecutable

Añadir el .exe generado con JSmooth.

Paso 5: Seleccionar idioma

Seleccionar el idioma de la aplicación.

Paso 6: Configurar instalador

  • Carpeta de salida: Output
  • Nombre del instalador: ElectricCityFactura_Instalador.exe

Paso 7: Personalizar aspecto

Modificar la apariencia del instalador según preferencias.

Paso 8: Configuración final

Desmarcar la casilla de compilación automática y finalizar.

El instalador se generará en la carpeta Output:

C:\Users\Antonio\Desktop\Cosas_de_curso\2_DAM\Interfaces\ProyectoFacturaLuz\Output\ElectricCityFactura_Instalador.exe

4. Instalación desatendida

Parámetros de instalación silenciosa:

Parámetro Descripción
/SILENT Muestra solo una barra de progreso
/VERYSILENT Completamente silencioso, sin ventanas
/SUPPRESSMSGBOXES Suprime cuadros de mensaje

Ejemplo de instalación desatendida:

C:\Users\Alumno\Desktop\Mario_PR51\Folder_aplicacion\Output>ElectroFactura_Instalador.exe /VERYSILENT /SUPPRESSMSGBOXES /SP- /NOCANCEL

Tu comando personalizado:

C:\Users\Antonio\Desktop\Cosas_de_curso\2_DAM\Interfaces\ProyectoFacturaLuz\Output>ElectricCityFactura_Instalador.exe /VERYSILENT /SUPPRESSMSGBOXES /SP- /NOCANCEL

5. Desinstalación y feedback

Inno Setup genera automáticamente un desinstalador junto con el instalador:

unins000.exe
unins000.dat

Ubicación del desinstalador:

El desinstalador se encuentra en la misma carpeta donde se instaló la aplicación, típicamente:

C:\Program Files\ElectricCityFactura\unins000.exe

El usuario puede desinstalar la aplicación mediante:

  • Ejecutar unins000.exe directamente
  • A través del Panel de Control de Windows (Agregar o quitar programas)

🔧 Comandos útiles

Generar JAR:

./gradlew jar

Ejecutar aplicación:

./gradlew run

Limpiar archivos generados:

./gradlew clean
